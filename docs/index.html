Current Direction

FBM maps
-> defect generation
-> multi-defect synthetic maps
-> multi-defect segmentation training and validation
-> local segmentation tool for real-data pattern asset extraction

WaferMap의 현재 목적은 모델을 먼저 키우는 것이 아니라, defect segmentation model을 만들 수 있는 라벨 데이터셋 제작 경로를 안정화하는 것입니다.

메인 UI는 repo 안의 local segmentation tool입니다. Pattern asset 저장 결과가 곧 synthetic dataset의 입력입니다.

End-To-End Flow

Manifest

raw wafer 또는 real-like sample을 real_unlabeled_manifest/v1로 정리합니다.

Tool

run_segmentation_tool.py로 wafer sample을 열고 family별 mask를 직접 작성합니다.

Assets

저장된 grade.png, mask.png, preview.png, metadata.json을 검수합니다.

QC

build_pattern_asset_report.py로 mask tightness와 family assignment를 확인합니다.

Synthesis

asset과 procedural fallback을 base wafer에 붙여 hybrid synthetic data를 만듭니다.

Readiness

asset_segmentation_manifest.csv와 smoke validation report를 생성합니다.

Key Commands

python scripts/run_segmentation_tool.py `
  --manifest configs/eval/real_unlabeled_synthetic_smoke.json `
  --sample-id real_like_synth_000000 `
  --assets-root data/pattern_assets

python scripts/run_pattern_asset_pipeline.py `
  --assets-root data/pattern_assets `
  --composed-dir data/synthetic/asset_composed `
  --work-dir outputs/pattern_asset_pipeline
