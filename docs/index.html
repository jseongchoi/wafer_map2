WaferMap 문서 표지판

FBM wafer map에서 대표 불량 패턴을 모으고, 라벨을 아는 합성맵을 만들어, multi-label U-Net segmentation 모델을 시작하는 프로젝트입니다.

대표 불량 패턴
-> manual asset / parametric rule
-> 합성 wafer map + 정답 mask
-> multi-label U-Net
-> 실제 wafer 예측
-> 사람이 수정
-> 더 좋은 asset으로 다시 축적

이 파일부터 보면 됩니다

핵심은 모든 wafer를 손으로 누끼 따지 않는 것입니다. 사람이 대표 패턴이나 규칙을 지정하고, 코드는 그 규칙을 full-size binary mask로 만들어 학습 데이터에 넣습니다.

이 HTML은 프로젝트의 메인 표지판입니다. 어떤 문서를 먼저 봐야 하는지, 지금 구현된 코드가 어디인지, 다음에 무엇을 해야 하는지 여기서 출발하면 됩니다.

한 장 요약:
1. 대표 불량 패턴을 모은다
2. 애매한 불량은 억지로 라벨링하지 않는다
3. ring/edge/shot_grid는 코드 기반 mask도 쓴다
4. 합성맵은 정답 mask를 자동으로 안다
5. 모델 예측을 실제 wafer에서 고쳐 다시 asset으로 쌓는다

먼저 읽을 순서

  1. 프로젝트 목적 확인: core_direction.md에서 이 프로젝트가 분류기가 아니라 segmentation dataset factory라는 점을 확인합니다.
  2. 전체 흐름 확인: end_to_end_workflow.md에서 raw wafer부터 prediction correction까지 명령 순서를 봅니다.
  3. 라벨을 어떻게 모을지: label_data_guidelines.md에서 bbox, mask, manual label, parametric label, mixed_unknown 예시를 봅니다.
  4. 학습 데이터 규격: training_data_contract.md에서 `arrays.npz`, `metadata.json`, manifest CSV, U-Net input/target channel을 봅니다.
  5. 실행할 때: operator_manual.md에서 실제 command와 troubleshooting을 봅니다.

질문결론예시
모든 불량을 손으로 mask 해야 하나?아니요.local blob은 사람이 lasso로 따도 되지만, shot_grid를 모든 die마다 칠하면 지옥입니다.
U-Net 학습의 진짜 정답은?family별 binary mask입니다.`bbox_xywh`는 crop/review용이고, `pattern_masks[family, y, x] = 1`이 학습 정답입니다.
shot마다 왼쪽 아래가 불량이면?parametric label로 저장하고 코드로 mask를 만듭니다.3x3 shot, affected slot `[2, 0]`, intra-die lower-left region을 지정합니다.
ring, edge, 긴 scratch는?가능하면 규칙 기반으로 mask를 만듭니다.ring은 center/radius/width, edge는 angle/radial band, scratch는 polyline+width.
합성맵의 역할은?라벨을 아는 training sample을 만드는 것입니다.asset을 어디에 붙였는지 아니까 local/scratch/ring mask를 자동으로 알 수 있습니다.
실제 wafer와의 차이는?prediction correction loop로 줄입니다.synthetic으로 첫 모델을 만들고, 실제 wafer 예측을 사람이 고쳐 real asset으로 다시 저장합니다.

불량별 라벨링 방식 예시

불량 유형사람이 할 일코드가 할 일학습 사용
`local` blobblob 주변을 brush/lasso로 칠함mask를 valid 영역으로 자르고 asset 저장`manual_mask`로 사용
`scratch`선을 따라 점 몇 개를 찍고 width 지정polyline 주변 tube mask 생성`parametric_mask` 또는 수동 mask
`ring`중심, 반지름, 두께, 각도 범위 지정annulus/arc mask 생성`parametric_mask`
`edge`몇 시 방향부터 몇 시 방향인지, edge 폭 지정edge sector mask 생성`parametric_mask`
`shot_grid`shot layout과 affected slot 지정반복 위치 mask 자동 생성`parametric_mask`
애매한 diffuse대략 region만 표시하고 이유 기록학습 제외, review 후보로 유지`mixed_unknown`, `weak_region`

애매한 걸 억지로 `scratch`나 `ring`에 넣으면 모델이 잘 배우는 게 아니라 헷갈리게 배웁니다. 애매하면 `mixed_unknown`으로 빼는 게 더 건강합니다.

학습 데이터는 이렇게 생깁니다

U-Net이 바로 읽는 최종 training sample은 사람이 그린 JSON이 아니라 rasterize된 `arrays.npz`입니다.

data/synthetic/asset_composed/
  asset_composed_000001/
    arrays.npz
    metadata.json

outputs/pattern_asset_pipeline/
  asset_segmentation_manifest.csv
arrays.npz 핵심:
severity             [H, W]
wafer_mask           [H, W]
valid_test_mask      [H, W]
stby_mask            [H, W]
pattern_masks        [7, H, W]
pattern_intensity    [7, H, W]

자세한 규격은 학습 데이터 규격 문서에 예시와 함께 정리했습니다.

주요 문서

라벨 데이터 가이드wafer 1장 기준으로 labels.json, masks 폴더, bbox와 mask 차이, parametric label 예시를 설명합니다. 학습 데이터 규격U-Net이 읽는 `arrays.npz`, `metadata.json`, manifest CSV, input/target channel 규격입니다. 모델 이해 가이드U-Net이 무엇을 입력받고, 어떤 mask를 예측하고, prediction을 어떻게 해석하는지 설명합니다. 유사맵 검색 가이드feature, mask, subtype, embedding으로 비슷한 wafer를 찾는 절차입니다. 불량 심각도 수치화mask feature에서 area, intensity, location, spread score를 계산하는 방법입니다. 전체 실행 흐름raw wafer부터 asset, synthetic dataset, U-Net, prediction correction까지의 command 흐름입니다. 작업자 매뉴얼실제로 명령을 실행할 때 보는 짧은 runbook입니다. 불량 family 정의local, scratch, ring, edge, shot_grid, random을 어떤 기준으로 나누는지 정리합니다. 합성 데이터 파이프라인pattern asset을 base wafer에 붙이고 readiness report를 만드는 과정입니다. 문서 품질 검증표문서 하나하나가 현재 목적과 맞는지 검토한 결과입니다.

  1. 실제 wafer에서 명확한 `local`, `scratch`, `ring`, `edge` 예시를 먼저 모읍니다. 애매한 것은 `mixed_unknown`으로 빼둡니다.
