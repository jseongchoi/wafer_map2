라벨 없는 실제 wafer 적용 준비 단계

현재 솔루션은 실제 wafer에서도 계산 가능한 feature를 뽑고, 그 feature로 비슷한 wafer를 찾은 뒤, 전문가 리뷰 결과를 다음 feature 보강 또는 AI 모델 후보로 연결하는 구조다.

현재 단계 Phase 4 주 경로 feature retrieval + 전문가 리뷰 다음 확인 real wafer 5~20장

현재 결론

1차 기준선 솔루션은 잡혔다. 아직 최종 확정은 아니며, 다음 확인 단계는 실제 wafer 전문가 리뷰다.

50 전체 유사 wafer 검색에 쓰는 compact feature 수
1.36x scale 155장 top-k retrieval lift
1.40x holdout 120장 top-k retrieval lift
58 현재 통과한 pytest 개수

대기

내가 지금 해야 할 일

  1. real wafer 5~20장을 익명 sample id로 고른다.
  2. severity, wafer_mask, valid_test_mask, stby_mask를 포함한 .npz를 export한다.
  3. manifest를 작성하고 sanity 결과를 먼저 확인한다.
  4. top-k neighbor 중 20~50 pair를 전문가 리뷰 template에 채운다.
  5. next_action_queue를 보고 feature 보강 또는 AI 모델 후보를 결정한다.

보안상 남기지 말 것

  • 실제 wafer raw image
  • 실제 wafer raw array .npz
  • 실제 file path가 들어간 manifest 원본
  • lot, tool, recipe, chamber, wafer id

AI 모델 현재 위치

지금 중심은 deep model이 아니라 feature 기반 검색과 전문가 리뷰다. Segmentation smoke test는 학습 배관이 연결되는지 확인한 수준이며, 실제 리뷰에서 scratch/local 같은 계열이 반복적으로 약할 때 모델 보강으로 넘어간다.

실행 명령

python scripts/extract_real_unlabeled_features.py `
  --manifest D:/secure_fbm/real_manifest.json `
  --reference-features outputs/reports/fbm_grouping_scale_features.csv `
  --features-out outputs/reports/real_unlabeled_features.csv `
  --sanity-out outputs/reports/real_unlabeled_sanity.json `
  --report-out outputs/reports/real_unlabeled_report.html `
  --neighbors-out outputs/reports/real_unlabeled_neighbors.csv `
  --review-template-out outputs/reports/real_unlabeled_expert_review_template.csv
