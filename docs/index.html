라벨 없는 실제 wafer 적용 준비 단계

현재 솔루션은 실제 wafer에서도 계산 가능한 feature를 뽑고, 그 feature로 비슷한 wafer를 찾은 뒤, 전문가 리뷰 결과를 다음 feature 보강 또는 AI 모델 후보로 연결하는 구조다.

현재 단계 Phase 4 주 경로 feature retrieval + 전문가 리뷰 다음 확인 real wafer 5~20장

처음 보는 용어와 변수

아래 용어는 이 프로젝트 문서와 CSV에 반복해서 나오는 핵심 단어다. 더 자세한 표는 용어와 변수 설명에서 확인한다.

severity wafer map의 Grade 0~7 값이다. wafer 밖과 stby 영역은 0으로 둔다.
wafer_mask wafer 내부와 wafer 밖을 구분한다. wafer 밖 0과 wafer 안 정상 Grade 0을 구분하는 데 필요하다.
valid_test_mask 실제 test가 수행된 위치를 표시한다. stby나 미측정 영역은 0이다.
stby_mask stby fail chip 영역을 표시한다. stby를 Grade 7 불량으로 오해하지 않게 한다.
manifest 실제 raw data를 repo에 넣지 않고, 보안 경로의 raw PNG 또는 `.npz`와 metadata를 참조하는 JSON 파일이다.
png_grayscale_raw 실제 8-bit grayscale PNG를 gray value 기준으로 Grade 0~7과 stby 후보로 변환하는 입력 형식이다.
top-k query wafer 하나에 대해 가까운 neighbor k개를 보는 방식이다. top-5면 가장 가까운 5개를 본다.
retrieval lift random 검색보다 nearest-neighbor 검색이 얼마나 나은지 보는 비율이다. 1보다 크면 random보다 낫다.
feature drift 새 wafer feature 분포가 reference와 얼마나 다른지 보는 sanity signal이다. 성능 점수가 아니다.
next_action_queue 전문가 리뷰에서 나온 실패 유형을 모아 다음 feature 보강이나 AI 모델 후보로 정리한 목록이다.

대기

내가 지금 해야 할 일

  1. real wafer 5~20장을 제품별 raw PNG 폴더로 준비한다.
  2. 8-bit grayscale 값이 0,31,151,175,191,207,223,255 기준인지 확인한다.
  3. batch script로 manifest와 sanity 결과를 생성한다.
  4. top-k neighbor 중 20~50 pair를 전문가 리뷰 양식에 채운다.
  5. next_action_queue를 보고 feature 보강 또는 AI 모델 후보를 결정한다.

보안상 남기지 말 것

  • 실제 wafer raw image
  • 실제 wafer raw array .npz
  • 실제 file path가 들어간 manifest 원본
  • 제품명, lot, tool, recipe, chamber, wafer id

AI 모델 현재 위치

지금 중심은 deep model이 아니라 feature 기반 검색과 전문가 리뷰다. Segmentation smoke test는 학습 배관이 연결되는지 확인한 수준이며, 실제 리뷰에서 scratch/local 같은 계열이 반복적으로 약할 때 모델 보강으로 넘어간다.

실행 명령

python scripts/analyze_png_raw_folders.py `
  --raw-root D:/secure_fbm/raw_png `
  --production-run `
  --geometry-json D:/secure_fbm/product_geometry.json `
  --out-dir outputs/reports/real_png_batch `
  --reference-features outputs/pre_real_readiness/reports/synthetic_reference_features.csv
